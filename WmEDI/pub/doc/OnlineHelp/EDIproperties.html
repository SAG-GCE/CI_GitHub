EDI Properties Settings

Updating properties on the Properties Settings page changes the properties in the EDI properties.cnf file and in memory, which results in the changes taking effect immediately. The properties.cnf file is located in the webMethods\IntegrationServer\packages\WmEDI\config directory, where webMethods\IntegrationServer is the directory in which your Integration Server is installed.

You can add properties not currently listed on the Properties Settings page. If the property you want to set is not listed, add it by making a new line on the Properties Settingspage. Be sure you use the correct case.

For a description of a property, click the link for the property or the property type in the following lists.

Functional Acknowledgment (FA) Property

EDIMaxFATransactionErrors

Description:

 The maximum number of errors that can be reported for any one FA transaction.

Possible Values:

 n, a positive whole number indicating the maximum number of errors for any one FA transaction.
  -1, indicating any number of errors can be reported for any one FA transaction.

Default Value:

 100, indicating that up to and including 100 errors can be reported for any one FA transaction.

Large Document Handling Property

EDIBigDocThreshold

Description:

 The threshold size at which the EDI Module should consider a document to be large.

Possible Values:

 -n, any negative whole number indicating that the EDI Module considers no documents as large. The EDI Module processes all documents in the traditional manner, reading the document content into memory during processing.
  0, indicating that the EDI Module considers all documents large. Because all documents that The EDI Module receives contain more than 0 bytes, the EDI Module uses large document handling for all documents.
  n, any positive whole number indicating the number of bytes over which the EDI Module considers a document to be large.

Default Value:

 -1, indicating that the EDI Module considers no documents to be large.

Example:

 If you want the EDI Module to consider all documents over 1,000,000 bytes to be large, set this property to 1000000:
  EDIBigDocThreshold=1000000

Miscellaneous Property

EDIencoding

Description:

 The type of character encoding that the EDI Module should use when converting an EDI document to a String.

Possible Values:

 UTF-8, ISO8859-1, etc.

Default Value:

 UTF-8
